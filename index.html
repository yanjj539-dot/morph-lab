SCROLL THROUGH THE PROCESS↓

FOUR-STAGE JOURNEY

From raw material to a working release.

The process stays quiet on purpose: observe the real inputs, structure the system, prototype the interaction, then ship the result.

  1. 01OBSERVE

    把材料看清楚

    先拆解真实内容、语气、视觉资产和使用场景，不急着套模板。

    • content audit
    • reference map
    • interaction notes
  2. 02STRUCTURE

    建立可以生长的系统

    把模型输出、界面层级、组件节奏和品牌语言整理成可执行结构。

    • information architecture
    • design tokens
    • component rhythm
  3. 03PROTOTYPE

    做出可运行的试验品

    用前端、WebGL、传感器或自动化流程快速验证交互是否成立。

    • live interface
    • motion test
    • device loop
  4. 04RELEASE

    收束成正式作品

    删除装饰性噪音，保留叙事、性能、可访问性和可部署结果。

    • production build
    • browser QA
    • deployment notes
02 / PRACTICEREAL OUTPUTS / RUNNING SYSTEMS

WHAT I ACTUALLY MAKE

不把 AI 当成视觉标签，而是把它接入明确的设计判断、可测试界面和可交付工作流。

01

AI visual systems

AI 视觉系统

为批量图像、人格视觉和品牌语气建立明确约束，让生成结果保持同一套审美判断。

02

Interactive websites

交互网站

把作品集、产品页和研究笔记做成可以浏览、可以部署、可以复用的数字界面。

03

Physical interaction prototypes

实体交互原型

用 ESP32、传感器、声音与光效，把情绪反馈和硬件行为接到真实体验里。

04

Agent-assisted workflow

Agent 辅助工作流

让模型参与整理、初稿、检查和交付，但保留人的方向判断和质量门槛。
